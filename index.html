Teardown Touring Cars
Spawn Fix

An unofficial compatibility fix for the Steam Workshop mod Teardown Touring Cars (AI Racing). This patch restores proper vehicle spawning behavior after newer Teardown versions introduced initialization timing issues.

View Repository

🔧 What this fixes

  • Spawning outside of the vehicle
  • Spawning under the map or in invalid locations
  • Races failing to start correctly
  • Vehicle assignment happening before initialization completes

Only AVF_AI_V3.lua has been modified.

📦 Installation

Locate your installed copy of Teardown Touring Cars (AI Racing).

Default location:

Steam\steamapps\workshop\content\1167630\2668638370\
  1. Open the scripts folder.
  2. Find the original AVF_AI_V3.lua.
  3. Make a backup of the original file.
  4. Replace it with the fixed version from the repository.
  5. Launch Teardown and start a race through the mod manager.

🧠 Why this exists

I encountered the same spawning issue while using the original mod. After investigating the Lua scripts, the problem was traced to AVF_AI_V3.lua.

This patch was created to restore the original mod's functionality without changing the rest of the mod.

🔍 Investigation

The issue was reproduced by testing the mod across multiple maps. The spawning failure occurred consistently and was not limited to a specific map.

Further testing showed that manually entering vehicles worked correctly, meaning the issue was isolated to the spawning process.

Reviewing the scripts revealed the use of setPlayerVehicle(). The issue appeared to be caused by newer Teardown versions returning a vehicle handle before the vehicle was fully initialized.

The fix delays the call until the vehicle becomes available. Once successful, the temporary check disables itself.
